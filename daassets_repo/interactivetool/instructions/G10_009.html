GIS data integration

  • Data integration means ‘putting together’ data and map layers.
  • Some important considerations when integrating data include data format, data cleaning and quality control.
  • Some examples of data formats are shape files, tiff, png and jpeg files.
  • Data cleaning involves editing and getting rid of ‘noise’ (data that distracts), checking for valid geometry and checking that attributes are correct.
  • Quality control ensures that data is entered into a GIS correctly in the first place. If inaccurate data is used that result will be that an inaccurate map is produced. Tests can be run on data to check its validity.
  • Data layers need to be at the same projection and co-ordinate reference system (CRS) when integrated.
  • A very important consideration for integrating spatial data sets and map layers is that they overlay correctly. Overlaying data is placing data on top of each other in layers in a similar way to which one would overlay cheese, tomato and lettuce on bread when making a sandwich.
