Data Standardisation, data security and data sharing

Data Standardization
In the 40 or so years since GIS started, many different data formats and communication protocols have been developed and used. Because there were so many formats in use, it used to be very difficult to share data. Nowadays, any good GIS software can read and write multiple different formats, and this allows easy, more effective sharing of data.

Data Sharing
It is a huge investment to capture and maintain high-quality data. GIS practitioners would never get their work done if they had to capture all the data they need themselves.

Data Security
For a government to work efficiently, departments need to share data rather than operate in isolation from each other. For example, all departments need the demographic data generated by the Census. Interoperability is a situation where diverse organisations and systems can share data and work together seamlessly.

Private companies might want to protect their data, in which case, they need to implement security measures. How an organisation implements security depends on its requirements.


Data security can also be implemented to various degrees:

  • read-only access
  • write or edit access
  • access that allows the user to delete data sets or create new ones.
