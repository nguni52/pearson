GIS data integration and identifying layers

To build up a GIS project, one of the first steps is to identify and acquire the data and map layers that you need. Once you have the data and map layers, the next step is to integrate them, which means ‘put them together’.

A very important consideration for integrating spatial datasets and map layers is that they overlay correctly. All the layers you need must firstly overlay or line up accurately on top of each other and they must be at the same projection.

For this to happen, these conditions need to be met:

  • The co-ordinate reference system (CRS) of the geometry must be described properly.
  • The GIS software must be able to understand the CRS that is used and project or transform the data ‘on the fly’ into a common CRS or else convert each data set permanently into a common CRS.

In South Africa, the CRS would normally be the South African Co-ordinate Reference System, which is based on the Transverse Mercator projection (also know as Gauss Conform or Kruger) and the Hartebeesthoek 1994 datum, using metres as the co-ordinate units.
